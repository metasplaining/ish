## Requirements

- GLOSSARY.md has entries for all 13 terms listed in the proposal.
- Existing `module` and `project root` and `visibility` entries are updated in place.
- New entries follow the existing table format with a `See [...]` reference where applicable.

## Tasks

- [x] 1. Update the **module** entry — `GLOSSARY.md`

  Current: "A unit of code organization. Every `.ish` file defines a module. The module path mirrors the file path. See [docs/spec/modules.md](docs/spec/modules.md)."

  New: "A `.ish` source file under `src/`. The module path is derived from the file path relative to `src/`, with `index.ish` mapping to the parent directory path. See [docs/spec/modules.md](docs/spec/modules.md)."

- [x] 2. Update the **visibility** entry — `GLOSSARY.md`

  Current: "All symbols default to `pub(self)` (visible only within the current module)." (in syntax.md, not yet in GLOSSARY)

  New entry: "One of `priv` (current module only), `pkg` (all project members, the default), or `pub` (external dependents). Applied as a keyword before `fn`, `let`, or `type` declarations. See [docs/spec/modules.md](docs/spec/modules.md)."

- [x] 3. Update the **project root** entry — `GLOSSARY.md`

  Current: "The directory containing `project.json`. Determined by walking up the directory tree from the source file. Defines the boundary for `pkg` visibility."

  If already correct, verify and leave. If different, update to match the above.

- [x] 4. Add **module path** entry — `GLOSSARY.md`

  "The `/`-separated identifier sequence used in `use` statements to name a module. Does not include the `src/` prefix. Example: `net/http` refers to `src/net/http.ish`. See [docs/spec/modules.md](docs/spec/modules.md)."

- [x] 5. Add **source root** entry — `GLOSSARY.md`

  "The `src/` directory under a project root. All importable modules are nested under it. The `src/` prefix is not part of any module path. See [docs/spec/modules.md](docs/spec/modules.md)."

- [x] 6. Add **project member** entry — `GLOSSARY.md`

  "Any file physically located under the project root. Project members receive `pkg` access and the project's dependency configuration. See [docs/spec/modules.md](docs/spec/modules.md)."

- [x] 7. Add **standalone script** entry — `GLOSSARY.md`

  "A file not under any `project.json`. Uses the installation default project, or a `bootstrap` directive for custom configuration. See [docs/spec/modules.md](docs/spec/modules.md)."

- [x] 8. Add **`use` directive** entry — `GLOSSARY.md`

  "The import statement for modules. Supports four forms: plain (`use foo/bar`), aliased (`use foo/bar as b`), selective (`use foo/bar { Type }`), and selective-with-rename (`use foo/bar { Type as T }`). See [docs/spec/syntax.md](docs/spec/syntax.md)."

- [x] 9. Add **`declare` block** entry — `GLOSSARY.md`

  "An anonymous, declarations-only grouping. Does not introduce a namespace. Allows mutually recursive declarations. Implicit declare wrapping is applied to files loaded via `use`. See [docs/spec/syntax.md](docs/spec/syntax.md)."

- [x] 10. Add **`bootstrap` directive** entry — `GLOSSARY.md`

  "A standalone script directive that provides the same configuration as `project.json` for a single file. Available in three forms: filesystem path, HTTPS URL, or inline JSON. See [docs/spec/syntax.md](docs/spec/syntax.md)."

- [x] 11. Add **interface file** entry — `GLOSSARY.md`

  "A `.ishi` file that declares the `pub` contract of a module. Generated by `ish interface freeze`. Enforced at compile time when present: all `pub` declarations in the `.ishi` file must match the implementation exactly. See [docs/spec/modules.md](docs/spec/modules.md)."

- [x] 12. Add **index module** entry — `GLOSSARY.md`

  "A module defined by `index.ish` in a directory. Its module path is the directory path, not `<dir>/index`. Example: `src/net/http/index.ish` defines module `net/http`. Having both `foo.ish` and `foo/index.ish` is a build error. See [docs/spec/modules.md](docs/spec/modules.md)."

- [x] 13. Add **installation default** entry — `GLOSSARY.md`

  "The default project used when no `project.json` is found in the directory hierarchy. Includes only the standard library. See [docs/spec/modules.md](docs/spec/modules.md)."
