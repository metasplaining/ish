## Inline Decisions Pending Incorporation

1. **Interface file generation (D4 revision):** Interface files are never generated automatically. The tool `ish interface freeze` or `ish interface freeze module_name` causes interface files to be generated. Interface files contain only the `pub` declarations from the associated `ish` file. The declarations must match exactly or an error is thrown at build time.

2. **`mod { }` → `declare { }` (D5 revision):** Eliminate `mod { }` blocks from the spec, and replace them with `declare { }` blocks. Declare blocks serve the same function that mod blocks did. They define a block of code that is declared together and is allowed to be cyclically referential. But declare blocks are anonymous and do nothing to the namespace. Additionally, only declarations are allowed in declare blocks. No top level command invocations or function calls are allowed. Commands and functions called from within functions being declared are still allowed. When a project is packaged or when a script uses another module in the same project, each of the files pulled into the compilation is implicitly wrapped in a declare block.

3. **Top-level package exports (open question):** Projects should be able to export symbols at the top level. User suggestion: `public.ish` is a special module that instead of being named `parent/public` is just named `parent`. Alternatives analysis requested.
