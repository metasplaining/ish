## Inline Decisions Pending Incorporation

**D16-name resolved — `bootstrap`:** The standalone dependency directive is named `bootstrap`.

**D16 scope expanded:** In addition to granting access to `pub` APIs, `bootstrap` does everything a `project.json` would do for a project: setting ish version, defining dependencies, setting standards and assurance levels, etc.

**D17 resolved:** An importable module MUST have a `.ish` extension — `use` won't find it otherwise. The interpreter is much more forgiving of scripts: any input passed to the ish interpreter is a script. Convention (unenforced): executable scripts have a shebang and no extension.

**D18 partially resolved:** Scripts intended for public use go in a `scripts/` directory. Scripts intended for internal use go in a `tools/` directory. Distribution of scripts is deferred.

**New structural decision:** Projects must have a `src` directory. Module files are nested under `src/`, but `src` is not part of the `use` statement that imports them.
