## Inline Decisions Pending Incorporation

1. **D15 resolved — `index.ish`:** The chosen option for top-level package exports is `index.ish`. A file named `index.ish` in directory `foo/` is assigned the module path `foo` (not `foo/index`). This works at any directory level in the project, not just the top-level directory. If both `foo.ish` and `foo/index.ish` exist, it is a build error for both.

2. **Escape mechanism not needed (Option A note):** The inline note on Option A's open question ("what if someone needs a module named `foo/public`?") was marked as "not needed." Since `index.ish` was chosen (not `public.ish`), the operative reserved filename is `index`. The same principle applies: there is no escape mechanism for files that happen to be named `index.ish` — the name is reserved.

3. **Cross-tree `project` directive (new open question):** An ish file can use the `project` directive to reference a project file located elsewhere on the directory tree. The proposal was asked to analyze: what does this do to visibility rules? Is it a feature or a bug? Are there security implications or merely technical contract ones?
