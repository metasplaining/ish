## Inline Decision Pending Incorporation

**`pkg` visibility scope clarified (D4 update):**

> `pkg` visibility includes visibility to other modules in the project and also to scripts in the project. Any script under the project directory qualifies. It is not required to be in the `scripts` or `tools` directory. However, visibility is restricted to script files actually contained in the project. Inline scripts, or a shell that changes directory to (or is launched from) the project directory do not qualify. An expected usage pattern is for a project to define many non-public modules with no public modules, then a short script contained in the project calls into those modules and becomes the executable entry point for the project.
